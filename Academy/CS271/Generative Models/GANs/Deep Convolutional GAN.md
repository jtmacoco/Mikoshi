- DCGANs are essentially the same as regular GANs, but they use convolutional layers instead of fully connected layers, making them better suited for image generation
- <span style="color:rgb(192, 0, 0)">Still using generator and discriminator </span>
## Generator
- Generates the image
- Input for generator is random noise
	- This is the latent space
	- The random noise is just random numbers
- Idea is to force the generator to understand a very complex latent space overall
- It's randomly sampling from all the space possible which is a lot of info
- <span style="color:rgb(192, 0, 0)">NO ENCODER SETTING IT UP</span>
- So the generator creates fake images which are passed into the discriminator
	- The discriminator outputs some numbers
- <span style="color:rgb(192, 0, 0)">The flipping of the labels is basically that when the generator outputs an image (THIS IS A FAKE IMAGE) we set that label to 1 which is real</span>
- More info in the problem section of training generator
	- We do this because the generator is trying to trick the discriminator
	- So if the discriminator predictions are closer to 1, that means the generator is doing a good job
	- When training the Generator we use label 1 for the fake images, so when the discriminator predicts something like 0.3(assuming 0-1) then the discriminator knows that the generator is trying to trick it so the loss for the generator goes up since the discriminator can tell it's a generated image
## Discriminator 
- Input get's real or fake images
- Then the discriminator needs to tell whether the image is fake or real like a game
- It knows which are fake and real by training the network
- When it first starts it had no understanding of anything
- <span style="color:rgb(192, 0, 0)">If fake label 0 if real label 1</span>

## Training GANS and DCGANS
- Rescale the data to the range [-1,1] so we want to use the tanh activation function on the final layer of the generator
- Use tanh because it provides stronger gradients than Sigmoid
- We are using a range from [-1,1] because using a pixels typical range 0-255 is to large
- The discriminator is going to use Sigmoid activation function
	- This is because we want the numbers to be either 0 (fake) and 1 (real), so the closer the discriminator output (predicted output) is to 1 the more confident it is that the input image is real
## Training DCGAN
- Train the discriminator with some real images and some generated by the generator
- Supervised learning: label 1 real images and 0 fake images
- Uses binary cross-entropy for the loss since we're comparing pixel by pixel real and fake which is logical
- <span style="color:rgb(192, 0, 0)">Alternate training so train discriminator then train generator</span>
	- This makes training fair
## Training Generator
- So basically the generator is going to generate image from some random noise which is literally just random numbers 
- Then it creates  some training images which is then passed into the discriminator
- <span style="color:rgb(192, 0, 0)">A very important point is that the generator images are labeled as 1 (so real images) note this is flipped technically </span>
	- The reason this is kind of flipped is because during the discriminator training the generated images are labeled as 0 or fake
	- The reason for this is because the generator is trying to trick the discriminator
	- So if the discriminator outputs a prediction closer to 1 then that means the generator is doing a good job at tricking the discriminator and its loss will be low
	- But if the discriminator is predicting closer to 0 that means the generator is doing a poor job at fooling the discriminator and its loss will be high

## Problem Discriminator
- So if one of the 2 networks overpower one another then the model's won't work, so won't generate any good images
- Discriminator can overpower the generator
	- The signal from the loss function becomes to weak to drive any meaningful improvements in the generator
- The discriminator learns to separate real images from fake images and the gradients vanish completely
- So basically the discriminator can be to harsh
## Potential solutions 
- If discriminator loss function collapses, we must weaken the discriminator
	- Increase the dropout layer rate in the discriminator
	- Reduce the learning rate of the discriminator (so discriminator learns slower while generator learns faster basically)
	- Reduce the number of convolutional filters in the discriminator
	- Add noise to the labels when training the discriminator
	- Flip the labels of some images at random when raining the discriminator (this is very harsh)

## Problem Generator
- Mode Collapse which can happen when we basically break the legs of the generator too much, so the discriminator is so bad that it doesn't give any meaningful information to the generator
	- Basically make the generator so bad that it can't understand real from fake which is bad
- <span style="color:rgb(0, 176, 80)">Mode collapse: </span>The discriminator is not powerful enough, so the generator tricks the discriminator with a small sample of nearly identical images. 
## Potential Solution
- Re-build the legs make it less crippled
