- Intra-Instruction Concurrency
- A single instruction is divided into different phrases or stages
	- Pipelining would be an example
- So at a microscopic level we can run multiple instructions (think assembly) at the same time which will allow for speed up.
- **<span style="color:rgb(255, 0, 0)">Note</span>**  this is down all at the hardware level so a program can write sequential code and will get free speed up from the hardware because of the design and instruction level programming
